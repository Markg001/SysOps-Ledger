## Step By Step Guide How to Setup Active Directory Domain Service on Windows Server 2022
## Introduction

In this post, we will learn how to setup active directory on windows server 2022.
The first step is to get windows server 2022 install on a server. It is very similar to server 2012 R2 / 2016 / 2019.
In my case, the server after OS installation is as follows.

<img src="/assets/img/posts/ad-setup/setup.png" alt="Windows Server initial setup configuration" class="shadow">

Now, we need to setup the pre-installation task like changing Computer name, IP Address, Time Zone, and other properties. After changing the computer name, the server needs to be restarted.

### My Server Setup is:
- **Machine Name:** WS2K22PDC  
- **IP Address:** 10.10.10.99  
- **DNS:** 10.10.10.99  
*Note: Since the server will act as DNS server, for DNS server field you can use local host address (127.0.0.1) or the servers’ IP address itself (10.10.10.99)*

<img src="/assets/img/posts/ad-setup/server-manager.png" alt="Server Manager dashboard" class="shadow">
### Active Directory setup process is divided into two major parts:
1. Install Active Directory Domain Service
2. Promote server as Domain Controller
3. DNS Configuration (Since we are using our Domain Controller server as a DNS server also, there is not
much benefit on using standalone internal DNS Server)

Let’s walk through the methods of installing active directory on windows server 2022 and adding domain in new forest.
### Step-1: Install Active Directory Domain Services (ADDS) Role
1. Need to login to your server with administrator account.
2. Search and Open Server Manager and then Dashboard.
3. Click on Add roles and features or you can also go through the Tools on Server Manager and then Add roles and features.
<img src="/assets/img/posts/ad-setup/3.png" alt="Add roles and features" class="shadow">
4. Click on Next to continue.
5. Select Role-based or feature-based installation and click on Next.
<img src="/assets/img/posts/ad-setup/role-based.png" alt="Role-based installation" class="shadow">
6. Since I am installing AD DS server role locally, I will select “**Select a server from the server pool”**. Select the server on which you want to install AD DS server role, in my case its WS2K22PDC. Click on Next.
<img src="/assets/img/posts/ad-setup/server.png" alt="Select server" class="shadow">
7. Now select Active Directory Domain Services role from the Select server roles page.
<img src="/assets/img/posts/ad-setup/roles.png" alt="Active Directory Domain Services role" class="shadow">
8. A pop-up will appear to add features that are required for Active Directory Domain Services. Click on Add Features and then click on Next.
<img src="/assets/img/posts/ad-setup/add-features.png" alt="Add features" class="shadow">
9. Now, the Active Directory Domain Services role has been selected. Again, select DNS server role and then click on add features to add required additional features. Click on Next.
<img src="/assets/img/posts/ad-setup/dns.png" alt="DNS server role" class="shadow">
10. Now, click on Next to continue.
<img src="/assets/img/posts/ad-setup/next.png" alt="Next" class="shadow">
11. On the Confirmation page, click on Install to start the installation of Active Directory Domain Services and DNS Server role.
<img src="/assets/img/posts/ad-setup/install.png" alt="Install" class="shadow">

12. The installation process will start and it will take few minutes to complete the installation. Once the installation is completed, click on Close.
 
#### Step-2: Promote Server as Domain Controller
1. After the installation of Active Directory Domain Services role, you will see a notification flag on Server Manager. Click on the notification flag and then click on Promote this server to a domain controller.
<img src="/assets/img/posts/ad-setup/promote.png" class="shadow">
2. On the Deployment Configuration page, select Add a new forest and then enter the Root domain name. In my case, I am using “**MrMark.local**” as my root domain name. Click on Next to continue.
<img src="/assets/img/posts/ad-setup/forest.png" alt="Add a new forest" class="shadow">
3. On the Domain Controller Options page, select the Forest functional level and Domain functional level. Since I am using windows server 2022, I will select windows server 2022 for both forest and domain functional level. Also, select the Domain Name System (DNS) server and then enter the Directory Services Restore Mode (DSRM) password. Click on Next to continue.
<img src="/assets/img/posts/ad-setup/domain-options.png" alt="Domain Controller Options" class="shadow">
4. On the DNS Options page, it will give warning about DNS delegation, but it can be ignored. click on Next to continue.
5. On the Additional Options page, it will automatically populate the NetBIOS domain name. You can change it if you want. Click on Next to continue.
6. On the Paths page, you can specify the location for the database, log files, and SYSVOL folder. By default, it will be installed on C drive, but it is recommended to change the location to another drive if you have multiple drives. Click on Next to continue.
<img src="/assets/img/posts/ad-setup/paths.png" alt="Paths" class="shadow">
7. On the Review Options page, review all the options that you have selected and then click on Next to continue. 
8. On the Prerequisites Check page, it will check for all the prerequisites for installing Active Directory Domain Services. If all the prerequisites are passed, then click on Install to start the installation process.
<img src="/assets/img/posts/ad-setup/prerequisites.png" alt="Prerequisites Check" class="shadow">
9. The installation process will start and it will take few minutes to complete the installation. Once the installation is completed, the server will automatically restart.
<img src="/assets/img/posts/ad-setup/reboot.png" alt="Restart" class="shadow">
After the restart, you can login to the server with the domain administrator account that you have created during the installation process.
<img src="/assets/img/posts/ad-setup/restart.png" alt="Restart" class="shadow">
### Step-3: DNS Configuration
1. After the installation of Active Directory Domain Services and DNS Server role, We can now see the domain name and the role AD DS and DNS installed in Server Manager page.
<img src="/assets/img/posts/ad-setup/manager.png" alt="Server Manager" class="shadow">
2. Now, We can now verify the Domain Controller configuration through Tools on Server Manager page. Open Active Directory Users and Computers from the Tools options.
<img src="/assets/img/posts/ad-setup/users.png" alt="Active Directory Users and Computers" class="shadow">
3. In the Active Directory Users and Computers page, you can see the domain name that we have created during the installation process. You can also see the default containers and organizational units (OUs) that are created by default.
<img src="/assets/img/posts/ad-setup/ous.png" alt="Organizational Units" class="shadow">

### We have successfully setup Active Directory Domain Services (Domain Controller) on Windows Server 2022.
