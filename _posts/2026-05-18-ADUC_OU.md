## Active Directory Users and Computers (ADUC) Organization Unit
What is Organization Unit?
    This is a container within Active Directory that can hold users, groups, computers, and other OUs. It is used to organize and manage these objects in a hierarchical structure. OUs can be nested within each other, allowing for a flexible and scalable way to manage resources in an Active Directory environment.
Benefits of using OUs:
1. Delegation of Control: OUs allow administrators to delegate specific permissions to manage the objects within them. This means that you can assign different levels of access to different administrators or groups, improving security and efficiency.
2. Group Policy Application: OUs can be used to apply Group Policies to specific sets of users or computers. This allows for centralized management of settings and configurations across the organization.
3. Organization and Structure: OUs help to organize resources in a logical manner, making it easier to manage and locate objects within Active Directory. This is especially beneficial in larger organizations with many users and resources.
4. Simplified Administration: By grouping related objects together in OUs, administrators can simplify the management of those objects. This can lead to more efficient administration and reduced chances of errors when managing users, groups, and computers.
5. Scalability: OUs can be nested, allowing for a hierarchical structure that can grow with the organization. This means that as the organization expands, new OUs can be created to accommodate new users, groups, and resources without disrupting the existing structure.
In summary, Organization Units in Active Directory Users and Computers (ADUC) provide a way to organize and manage users, groups, computers, and other resources in a hierarchical structure. They offer benefits such as delegation of control, Group Policy application, improved organization, simplified administration, and scalability.

If you created the AD user in the wrong spot that means they could get the wrong set of Group Policy security settings.
3. So, I am going to right-click on the domain name ad.serveracademy.com and then select New and then select Organizational Unit.
4. In the New Object - Organizational Unit dialog box, type "Users" in the Name field or the name of the OU you want to create and then click OK.
5. The new OU named "Users" will now appear under your domain in the ADUC console. You can now create user accounts, groups, and other objects within this OU as needed.
